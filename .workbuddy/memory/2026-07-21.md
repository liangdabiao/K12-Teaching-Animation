## 重要修正（核查时发现早前"全部通过"读数不可靠，已修复）

重新抽查时发现两类问题，导致此前"40/40 全通过"的结论不可靠：

1. **生成器 build() 与 HEAD 占位符不匹配**：`gen_chem.py` 的 `build()` 用 `.replace("__TITLE__",...)` 形式，
   但 `HEAD` 仍写的是 `%(title)s` / `%(css)s` / `%(desc)s` / `%(legend)s` 形式 → 占位符从未被替换。
   后果：4 个生成器（chem/physics/bio/it，后三者 `from gen_chem import build`）产出的**全部 40 个文件**
   在可见的标题/说明/图例处显示字面量 `%(title)s` 等（HTML 文本 bug，不影响 JS 语法）。
   此外 浓硫酸稀释 的磁盘文件是更早一版（粉色主题、`A` 对象），其 `sceneReset` 被写成 `{{t:0,...}}`，
   触发 `Unexpected token ':'` 的**真实 JS 语法错误**。
2. **校验脚本计数器 bug**：`phase2_validate.js` 早期 `fail` 未自增，读数虚高/虚低，掩盖了上述问题。

### 修复动作
- 将 `gen_chem.py` 的 `HEAD` 两处 `%(title)s`、以及 `%(css)s/%(desc)s/%(legend)s` 全部改为 `__TITLE__/__CSS__/__DESC__/__LEGEND__`，与 `build()` 对齐。
- 修正 `phase2_validate.js` 计数器，并重跑 4 个生成器 + 全量校验。

### 最终复核（本次，可信）
- 全工作区动画页 `%(...)` 字面量残留：**0**
- Phase 2（40 个重生成文件）语法 `node --check`：**PASS=40 FAIL=0**
- 全库 173 个动画页语法：**0 失败 / 0 缺脚本**
